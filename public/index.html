FAQ

Have a question? No worries.


### How do lobbies work?

Once you purchase your plan, all lobbies will immediately be accessible. For instance, if you purchase 1000 lobbies, then you can access lobby/1, lobby/2, lobby/3, ..., lobby/1000.

### How are username and account information being handled?

You can create as many "usernames" as you want. This is because Sandcastle does not permanently store user information. It's up to you on how to handle usernames and account information.

Many application stores should allow players to backup local information to the cloud. If you can, proceed with that option.

The only thing we require is that you provide a player ID. The player ID must be a positive integer. It should be unique. One way to make it unique is to use timestamps. The player's ID is the exact time that they launched their game for the first time. For instance, if they launched their game on the 15th of October, 2022 11:30:45.367, then their player ID could be 15102022113045367. You can also use Unix Epoch.

We plan on releasing an option for storing account information later on, especially for web-based games.

### Are passwords and email required?

Sandcastle not only wants to limit the barrier to entry for you, but also for the player as well. Having to memorize a password and entering user email is too cumbersome.

With that said, plans for adding password and email is on the horizon with the ability to store account information.

### How much data can I send?

JSON data is capped at 1 kilobytes (around 1000 characters). It's beneficial to keep data small as some players might not have good Internet.

### What data can I send?

Anything that is compatible with JSON.

### What about leaderboards?

Leaderboards coming soon! The plan is that each game will be given a leaderboard that can hold 1000 accounts and can be reset monthly, yearly, or biyearly.

### How about sending and receiving files and images?

They're not supported at the time. If you just need profile pictures, your application store might have an API that provides the URL of profile pictures. One creative way is to use emojis for profile pictures.

One workaround is to host your own files and images and send the URLs instead.

### What about attackers?

You'll have the option to create a token that can be sent with the JSON.

For open source or web games, an option to block abusive IP addresses will be available later on.

### Isn't HTTP slower than TCP?

With the acceleration of fiber-optics, 5G, faster processors, and more efficient storage, using HTTP is more feasible than ever.

Our servers can process information up to 1/100 of a second on HTTP with ideal Internet, but real world speed will be slower depending on the player's connection.
